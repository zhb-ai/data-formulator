### 修复：Excel 日期类型单元格导致 React 渲染崩溃 (error #31)

**问题现象**：上传包含日期列的 Excel 文件时，页面报错 `Minified React error #31; args=[object Date]`，整个应用崩溃。

**根本原因**：ExcelJS 解析 Excel 单元格时返回原生 JavaScript `Date` 对象，而 React 无法将 `Date` 对象作为子节点渲染。该问题在数据管道的多个层级均缺少防护。

**涉及文件与修复内容**：

| 文件 | 问题 | 修复方式 |
|------|------|----------|
| `src/data/utils.ts` | `loadBinaryDataWrapper` 中 `cell.value` 直接存储 Date/富文本/公式等 ExcelJS 复杂类型 | 新增 `resolveExcelCellValue()` 函数，统一将 Date 转 ISO 字符串、富文本合并为纯文本、公式取结果值、超链接取文本 |
| `src/data/types.ts` | `coerceDate()` 函数直接 `return v`，Date 对象透传不做任何转换 | 改为 `v instanceof Date ? v.toISOString() : v`，确保 Date 对象不会泄漏到渲染层 |
| `src/views/ReactTable.tsx` | 无 `format` 时直接渲染 `{value}`，Date/Object 类型触发 React error #31 | 增加 `typeof value === 'object'` 判断，对象类型自动 `String()` 转换 |
| `src/views/SelectableDataGrid.tsx` | 同上，无 `format` 时 `{cellValue}` 直接渲染 | 同上，增加对象类型安全渲染 |
| `src/views/DataLoadingThread.tsx` | `format: (v) => v` 原样返回，不做任何转换 | 改为对象类型自动 `String()` 转换 |

**防御深度**：修复采用多层防御策略：
1. **数据源层**（`resolveExcelCellValue`）：在 Excel 解析时就将复杂类型转为基本类型
2. **类型系统层**（`coerceDate`）：作为兜底，确保 Date 对象在类型推断后被转为字符串
3. **渲染层**（ReactTable / SelectableDataGrid / DataLoadingThread）：即使前两层遗漏，渲染时也不会因对象类型崩溃

**问题现象**：从匿名分析 chart 对话时切换到登录状态，再次访问缓存数据，页面报错 `Cannot read properties of undefined (reading 'model')`，整个应用崩溃。

- `SupersetCatalog.tsx` 中数据集卡片右侧操作使用的是“图标 + 文案”的竖排 `Button`
- `SupersetDashboards.tsx` 中仪表盘数据集行右侧操作使用的是纯图标 `IconButton + Tooltip`
- 两个入口同属 Superset 数据加载，但交互样式不一致，视觉上割裂

**最终折中方案**：
- 统一采用 `Tooltip + IconButton` 风格
- 原因：
  1. 仪表盘面板在展开行内，横向空间更紧，图标按钮更合适
  2. 数据集面板改成同样风格后，卡片主体内容可获得更大展示空间
  3. 通过 Tooltip 保留操作语义，不牺牲可理解性

**涉及文件与修改内容**：

| 文件 | 修改内容 |
|------|----------|
| `src/views/SupersetCatalog.tsx` | 将每个数据集卡片右侧两个操作从竖排 `Button` 改为横排 `Tooltip + IconButton`：保留“加载（覆盖同名）”和“创建新数据集”两个动作，但不再显示常驻文案 |
| `src/views/SupersetDashboards.tsx` | 保持已有 `Tooltip + IconButton` 风格，作为统一后的基准样式 |

**升级到 0.7 时的注意事项**：
- 如果 `0.7` 中 Superset 数据集面板仍沿用文字按钮，需要同步改回 `Tooltip + IconButton`
- 这是纯前端 UI/UX 一致性调整，不依赖后端接口，可直接迁移

### 2. Superset 仪表盘移除“创建新数据集”按钮

**问题判断**：
- 在仪表盘数据集入口中，“按条件加载”本身已经支持生成新表名
- 因此额外保留一个“创建新数据集”按钮会和筛选加载能力重叠，增加认知负担

**涉及文件与修改内容**：

| 文件 | 修改内容 |
|------|----------|
| `src/views/SupersetDashboards.tsx` | 移除数据集行右侧最后一个 `AddIcon` 按钮 |
| `src/views/SupersetDashboards.tsx` | 同时清理相关无用状态：`suffixDialogOpen`、`suffixDialogDs`、`suffixInput` |
| `src/views/SupersetDashboards.tsx` | 删除整个 suffix dialog 相关 JSX 与无用 import（`AddIcon`、`Dialog`、`DialogTitle`、`DialogContent`、`DialogActions`、`Button`） |

**升级到 0.7 时的注意事项**：
- 如果 `0.7` 的仪表盘数据集行还保留独立“新建数据集”入口，应评估是否与筛选加载重复；当前 0.6 fork 的结论是“应移除”
- 回迁时注意不仅删除按钮，还要同步清理 suffix dialog 相关状态和 import，避免残留死代码

### 3. External Data Loaders / Local DuckDB 左侧表名悬浮提示

**问题现象**：
- 数据加载完成后，左侧表列表中的表名在窄面板里会被省略号截断
- 用户只能看到部分名称，不利于区分长表名或带后缀的表名

**解决方式**：
- 为左侧表列表中被截断的表名增加悬浮提示
- 鼠标停留时显示完整表名
- Tooltip 位置设为右侧（`placement="right"`），并增加 `enterDelay={400}`，减少快速划过时的干扰

**涉及文件与修改内容**：

| 文件 | 修改内容 |
|------|----------|
| `src/views/DBTableManager.tsx` | 给 `Regular Tables` 区域中的表名 `Typography` 外层增加 `Tooltip title={t.name}` |
| `src/views/DBTableManager.tsx` | 给 `Derived Views` 区域中的表名 `Typography` 外层增加同样的 `Tooltip` |

**升级到 0.7 时的注意事项**：
- 如果 `0.7` 中左侧表列表仍使用 `textOverflow: 'ellipsis'` 截断表名，需一并补回 Tooltip
- 这是一个小改动，但很容易在升级时遗漏，因为只涉及局部 JSX 包裹而非业务逻辑
